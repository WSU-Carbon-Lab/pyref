Navigator — Screen Stack
pub struct Navigator {
    // current screen + back-history
    stack: Vec<ScreenState>,
    forward: Vec<ScreenState>,

    // shared catalog handle
    pub catalog: CatalogHandle,
}

pub enum ScreenState {
    // new: file-explorer at any depth
    Explorer(ExplorerState),

    // adapted: was Screen::Beamtime
    Beamtime(BeamtimeState),

    // new: layout-config modal
    ConfigModal(ModalState),
}

impl Navigator {
    pub fn push(&mut self, s: ScreenState) {}
    pub fn pop(&mut self) {}
    pub fn current(&self) -> &ScreenState {}
    pub fn current_mut(&mut self) -> &mut ScreenState {}
}

Screen State Ownership

Navigator owns Vec<ScreenState> (back stack — no cloning, push/pop) │ ├─▶ ScreenState::Explorer(ExplorerState) │ │ │ ├─ level: ExplorerLevel Root | Experimentalist | Generic │ ├─ root: PathBuf DATA/ passed from CLI │ ├─ current_dir: PathBuf where we are now │ ├─ entries: Vec<DirEntry> enriched with metadata │ └─ list_state: ListState │ ├─▶ ScreenState::Beamtime(BeamtimeState) │ │ │ ├─ beamtime_path: PathBuf filter key into DATA catalog │ ├─ experimentalist: String for display + catalog filter │ └─ [existing fields from App] samples, tags, scans, groups… │ └─▶ ScreenState::ConfigModal(ModalState) │ ├─ depth_to_beamtime: u8 user-configured depth ├─ pattern: String regex / format string └─ return_to: Box<ScreenState> screen to resume after confirm CatalogHandle shared via Arc<Mutex<Connection>> or per-call open() ├─▶ list_experimentalists(root) ├─▶ list_beamtimes(root, experimentalist) ├─▶ query_scan_points(root, beamtime_path, filter) adapted from existing └─▶ catalog_status(path) → Indexed | Stale | Missing

Catalog Schema Addition

New columns added to bt_beamtimes (no breaking changes)
-- existing bt_beamtimes already has: id, beamtime_path, user, year, month, esaf_number, label
-- additions:
ALTER TABLE bt_beamtimes ADD COLUMN experimentalist  TEXT NOT NULL DEFAULT '';
ALTER TABLE bt_beamtimes ADD COLUMN data_root        TEXT NOT NULL DEFAULT '';
-- data_root: the DATA/ path this beamtime was ingested under
-- experimentalist: the folder name one level above the beamtime

-- new index for explorer queries
CREATE INDEX IF NOT EXISTS idx_bt_experimentalist
    ON bt_beamtimes(data_root, experimentalist);
Migration strategy: both new columns have DEFAULT '' so existing catalog DBs open without errors. The explorer_query functions treat empty strings as "unset" and fall back to path parsing to infer experimentalist from the beamtime_path. Re-ingesting any beamtime populates them correctly.
