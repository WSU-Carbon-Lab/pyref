Design Section 3 — Beamtime Screen Adaptation + Auto-Ingest

How BeamtimeState extracts from App, how queries shift to the DATA catalog, and what auto-ingest on entry looks like.

// BEFORE — per-beamtime catalog, no filter
let db = resolve_catalog_path(&beamtime_dir);

let db = data_root.join(".pyref/catalog.db");
let rows = query_scan_points(

pub fn query_scan_points(
    db:            &Path,

Migration path: query_files() (old per-beamtime API) is kept but deprecated. It still works against a per-beamtime DB. Callers are migrated one by one.

Esc during ingest: cancels the background thread (via a CancellationToken-style atomic flag), pops the BeamtimeState off the Navigator stack, and returns to the Explorer. Any partial writes to the catalog are left in place — a second entry attempt will resume indexing from where it stopped (already-indexed files are skipped by mtime check).

Navigator Entry Flow — Explorer → Beamtime

User presses [→] on a Beamtime row in ExplorerState │ ▼ navigator.enter_beamtime(path, experimentalist, data_root) │ ├──▶ CatalogStatus::Indexed │ │ │ ▼ │ BeamtimeState::load(data_root, beamtime_path) query DATA catalog │ │ │ ▼ │ Navigator.push(ScreenState::Beamtime(state)) immediate │ ├──▶ CatalogStatus::Stale │ │ │ ▼ │ BeamtimeState::new_ingesting(…) push with LoadingState::IngestingDirectory │ spawn ingest thread → ingest_beamtime(data_root, beamtime_path) │ └──▶ CatalogStatus::NotIndexed (same as Stale — auto-ingest) │ ▼ BeamtimeState::new_ingesting(…) spawn ingest thread → ingest_beamtime(data_root, beamtime_path) Ingest completes (try_recv_ingest in run loop): │ ├──▶ Ok(()) ──▶ BeamtimeState::reload_from_catalog() now queries DATA catalog │ ──▶ loading_state = Idle profiles appear │ ──▶ update ExplorerState entry → CatalogStatus::Indexed via shared channel │ └──▶ Err(e) ──▶ set_app_error(ingest_failed(…)) ──▶ loading_state = Idle error modal shown, [Esc] goes back

Back Navigation — Beamtime → Explorer

What happens on [←] / [h]
navigator.pop() — discards BeamtimeState
Restores the ExplorerState that was underneath
Cursor position in the explorer is preserved
(the ExplorerState was never mutated while Beamtime was active)

The Explorer's status cell for the beamtime just entered is updated to ✓ Indexed via a lightweight mpsc message sent when ingest completes.

No reload of the full explorer entry list needed.
Config saved on BeamtimeState drop
The same selections that are currently saved on quit (last_sample, last_tag, last_scan) are written to TuiConfig when the BeamtimeState is popped — not just on app exit.

This means returning to the same beamtime later in the same session (or on next launch with the same path) restores your previous cursor position.
