Current Launcher — Limitations

  • Flat list — no hierarchy, just recently-ingested paths
  • Global index at ~/.pyref/ — not co-located with DATA
  • No browsing — must type a path or pick from history
  • No metadata — no size, no count of beamtimes per experimentalist
  • No catalog context — no indication of what's indexed vs. not

Proposed Explorer — What Changes

  • Rooted at DATA/ — passed as CLI arg, full NAS awareness
  • Standard columns — Name, Date Modified, Size, Kind, # Beamtimes
  • Catalog indicator — shows whether .pyref/catalog.db exists at this level
  • Navigate down → Experimentalist → Beamtime → scan profiles
  • Inline ingest — press i to ingest selected folder

Current — No Intermediate Layer

  • Must already know the exact beamtime path to navigate to it
  • No way to see all beamtimes for a given experimentalist in one view
  • Cannot bulk-ingest multiple beamtimes from a single screen
  • The Beamtime screen works well — but it is the only screen

Proposed — Experimentalist Level

  • Per-beamtime catalog status — ✓ indexed / ⚠ stale / ✗ not indexed
  • FITS file count surfaced without opening the directory
  • i ingests selected beamtime, I ingests all not-yet-indexed
  • Entering a beamtime transitions to the existing Beamtime screen — unchanged
  • Non-beamtime dirs (scratch, misc) shown but clearly typed as generic

Navigation Flow Comparison

Current Navigation
▪ Launcher — flat list of past beamtime paths
   ↓ [↵] open selected or [o] type path
▪ Beamtime Screen — samples/tags/scans/profiles
   ↓ [h] nav_up goes to parent dir (reloads beamtime screen)
▪ No awareness of DATA root or experimentalist level
▪ Cannot navigate from Beamtime → sibling Beamtime
▪ Global beamtime index lives in ~/.pyref/
Proposed Navigation
▪ Explorer Screen — rooted at DATA/ (CLI arg)
   ↓ [→] enter dir — shows experimentalist dirs
▪ Experimentalist Level — beamtime folders with metadata
   ↓ [→] enter beamtime — if cataloged, goes to beamtime view
▪ Beamtime Screen — existing samples/tags/scans view (unchanged)
   ↓ [←/h] back to experimentalist level
▪ Full back/forward history stack across all levels
▪ Catalog lives at DATA/.pyref/ — co-located with data
