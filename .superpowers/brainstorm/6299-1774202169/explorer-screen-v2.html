Design Section 2 (revised) — Per-Experimentalist Layout Policy

Layout configuration is resolved per experimentalist, not globally. The root explorer flags unresolved folders. The modal is scoped to one folder at a time.

Data Model — NasLayout replaces LayoutPolicy

NasLayout — per-experimentalist map
/// Persisted to DATA/.pyref/layout.toml
pub struct NasLayout {
    /// default beamtime pattern used when
    /// an experimentalist has no override
    pub default_pattern: Option<String>,
    pub default_depth:   u8,  // default: 2

    /// one entry per depth-1 folder name
    pub experimentalists: HashMap<String, ExptPolicy>,
}

pub enum ExptPolicy {
    /// accepted by heuristic or left as
    /// default — no user action needed
    Auto,

    /// user specified a custom pattern
    /// and/or depth for this folder
    Custom {
        pattern: Option<String>,
        depth:   u8,
    },

    /// user marked as not an experimentalist
    /// (metadata dir, scratch, etc.)
    /// shown dimmed, never ingested
    Ignored,
}
Resolution states for a depth-1 folder
/// Computed at render time for each
/// depth-1 entry in the explorer
pub enum ExptResolution {
    /// heuristic matched, or user set Auto
    Resolved,

    /// heuristic failed, needs user action
    /// shown with ⚑ flag in explorer
    NeedsResolution,

    /// user marked as not an experimentalist
    Ignored,
}

/// DirEntry gains one new field:
pub struct DirEntry {
    // ...existing fields...
    pub expt_resolution: Option<ExptResolution>,
    // Some only when kind == Experimentalist
}
What counts as an experimentalist folder? Any depth-1 entry under DATA/ that is a directory and whose name does not start with "." — exactly as you described. The .pyref/ catalog dir, .DS_Store, etc. are automatically excluded by the leading-dot rule.

Resolution Flow

Navigator pushes Explorer(data_root) │ ▼ load_entries(data_root) read_dir + classify depth-1 folders │ ├──▶ name starts with "." ──▶ skip entirely (not shown) │ ├──▶ policy = Ignored ──▶ shown dimmed, ExptResolution::Ignored │ ├──▶ policy = Auto or Custom ──▶ ExptResolution::Resolved │ └──▶ policy missing (new folder) │ ▼ detect_policy(folder) shallow scan of subdirs │ ├──▶ confident ──▶ auto-set ExptPolicy::Auto → Resolved │ └──▶ not confident ──▶ ExptResolution::NeedsResolution ⚑ flagged User presses [r] on flagged row: │ ▼ Navigator.push(ConfigModal { target: "Chen", return_to: current_explorer }) │ ├──▶ Custom pattern confirmed ──▶ save to layout.toml, pop modal, refresh entries ├──▶ Ignored confirmed ──▶ save to layout.toml, pop modal, dim row └──▶ Global default accepted ──▶ set Auto, pop modal, mark Resolved
Persistence: Every confirmed resolution writes to DATA/.pyref/layout.toml immediately. On next startup the file is loaded first, so already-resolved folders never trigger the modal again. New experimentalist folders that appear on the NAS after the fact go through the same detection → flag → resolve cycle.

Entering a Flagged Folder

⚑ NeedsResolution
Pressing → or ↵ on a flagged row opens the resolution modal immediately instead of entering the folder.

The user must resolve before they can browse the folder's contents.
✓ Resolved
Normal entry — pushes a new ExplorerState scoped to that folder, using that experimentalist's policy to classify its subdirs as beamtime vs. generic.
✗ Ignored
Entry is still allowed — the user can browse a folder marked ignored. It's just excluded from beamtime counts and bulk ingest. Press x again to un-ignore.
