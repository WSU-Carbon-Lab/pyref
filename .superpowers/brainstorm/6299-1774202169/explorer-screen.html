LayoutPolicy — Heuristic + Override

LayoutPolicy struct
pub struct LayoutPolicy {
    /// folders at this depth from data_root
    /// are classified as Beamtime (default: 2)
    pub beamtime_depth: u8,

    /// regex applied to folder names to
    /// confirm Beamtime classification
    /// None = accept any at beamtime_depth
    pub beamtime_pattern: Option<Regex>,

    /// whether policy came from heuristic
    /// or from user config modal
    pub source: PolicySource,
}

pub enum PolicySource {
    Heuristic,   // auto-detected
    UserConfig,  // set via modal
    Default,     // fallback (depth=2)
}

// Default pattern — matches ALS-style dates
// and common beamtime naming conventions
const BEAMTIME_PATTERNS: &[&str] = &[
    r"^20\d{2}(Jan|Feb|Mar|...|Dec)$",  // 2024Jan
    r"^20\d{2}-\d{2}(-\d{2})?$",        // 2024-01
    r"^20\d{6}$",                        // 20240115
    r"(?i)beamtime",                     // contains "beamtime"
];
Classification logic (heuristic.rs)
pub fn classify_entry(
    path: &Path,
    data_root: &Path,
    policy: &LayoutPolicy,
) -> EntryKind {
    if path == data_root {
        return EntryKind::DataRoot;
    }
    let depth = relative_depth(path, data_root);
    match depth {
        1 => EntryKind::Experimentalist,
        d if d == policy.beamtime_depth as usize => {
            let name = folder_name(path);
            if policy.pattern_matches(name) {
                EntryKind::Beamtime
            } else {
                EntryKind::Directory
            }
        }
        _ => EntryKind::Directory,
    }
}

Heuristic trigger: On startup, detect_policy(data_root) scans the top 2 levels of the NAS tree (no deep walk). If ≥50% of depth-2 folders match a date pattern, it sets that pattern as the policy and marks confident=true. If <50% match — or no folders exist at depth 2 — it marks confident=false and pushes the ConfigModal before showing the explorer.

Modal behaviour: The preview pane updates live as the user types in either field — no confirm needed to see the effect. The regex is compiled on every keystroke; if invalid, the pattern field turns red and the preview shows the last valid state.
